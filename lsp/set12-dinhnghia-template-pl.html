❓ Quy tắc — Định nghĩa template phiếu lương

1. Template áp dụng cho tất cả dòng lương

1 template PL = 1 layout A4 cố định. Khi NV có N dòng lương (kiêm nhiệm BPTL, làm 2 ca…), engine dùng cùng 1 template render N lần, sinh ra N trang A4 độc lập. Mỗi trang chứa data per-row của dòng đó.

2. Cross-row BH/Thuế

BHXH/BHYT/BHTN và Thuế TNCN theo luật phải trích/tính 1 lần/kỳ — engine set7 đã handle: gán vào dòng có HT_HeSoBHXH MAX (thường là dòng HĐLĐ chính), dòng phụ = 0. Template không cần biết logic cross-row — chỉ hiển thị đúng field trong record.

3. Fallback template thiếu file

Nếu File template trống hoặc file không tồn tại → engine dùng template mặc định (checkbox Đặt làm template mặc định ở form). Nếu không có template default → hiển thị lỗi "Chưa cấu hình template" ở ESS/Mobile.

4. Nguồn dữ liệu tính

  • Millennium (core): dùng tham số Millennium (A.*, E.*, HR.*, PR.*) — mặc định cho các dự án khác MPHG.
  • MPHG LSP (engine mới): bật thêm tab "Tiêu chí MPHG LSP" (Cột grid set7 / HT_ set2 / TT_ set3) trong màn Gán CT. Chỉ tenant có role MPHG_PAYSLIP_ENGINE mới thấy option này.

5. Legacy migration

Record cũ không có nguonDL → default = MILLENNIUM. Zero-impact với các dự án đang chạy.

6. Kênh phân phối phiếu lương

  • HR bulk export: HR xuất PDF/Excel hàng loạt từ set7.
  • ESS self-view: NV xem/tải phiếu qua Web Dịch vụ cá nhân.
  • Mobile view: Mobile app MPHG với biometric auth.

Chi tiết follow-up ESS/Mobile → set7 › Follow-up.
