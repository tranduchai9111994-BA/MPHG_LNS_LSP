❓ Quy tắc — Điều kiện lọc phiếu lương mặc định

1. Filter SQL apply per-row

Điều kiện lọc được áp dụng trực tiếp trên bảng SalaryResult (kết quả tính từ set7). Mỗi row match filter = 1 phiếu lương = 1 trang A4. NV có N dòng match filter → sinh N phiếu. Không dùng GROUP BY nữa.

2. Cột trái (Tiêu chí khả dụng)

  • Template Millennium (core): chỉ hiển thị tiêu chí Millennium (A.*, E.*, PR.*).
  • Template MPHG_LSP: hiển thị thêm tiêu chí MPHG (C##, HT_*, TT_*) — chọn từ combo lọc "Nhóm nguồn" nếu muốn filter chỉ 1 loại.
  • Tiêu chí đã chọn (cột phải) tự động ẩn khỏi cột trái để tránh chọn trùng.

3. Toán tử & Liên kết AND/OR

Toán tử support: = ≠ > < ≥ ≤ LIKE IN IS NULL NOT NULL. Điều kiện đầu tiên tự động là WHERE, các điều kiện tiếp theo có combo AND / OR. Không có nesting (all-flat) — nếu cần logic phức tạp, dùng module BI riêng.

4. Kỳ lương áp dụng

  • Template MPHG_LSP → default kỳ = MPHG_yymm (từ set5).
  • Template Millennium → default kỳ = MILL_yymm (chuẩn core).
  • BA có thể override thủ công qua combo.

5. Preview SQL

Nút 👁 Preview SQL mở modal hiển thị WHERE clause đã build. Đây là mockup dựng SQL đơn giản để BA/QA thẩm định logic OR/AND — không phải SQL production thật.

6. Lưu & áp dụng

Filter này là mặc định của template. Khi HR phát hành phiếu, sẽ được apply tự động; HR có thể override tại thời điểm phát hành.
