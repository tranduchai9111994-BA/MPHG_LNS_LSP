📋 Quy tắc — Danh mục Function scalar
  1. Prefix chuẩn theo domain: HR_fn* (nhân sự), PTS_fn* (chấm công), PR_LSP_fn* (lương sản phẩm), PR_fn* (lương chung), PIT_fn* (thuế TNCN).
  2. Function Scalar KHÔNG return bảng. Cần trả nhiều cột/dòng thì convert sang iTVF (inline) hoặc mTVF (multi-statement). iTVF ưu tiên vì optimizer inline được.
  3. Tên params phải đồng bộ Danh mục Param hệ thống (set0): @EmpID, @FromDate/@ToDate, @PRMonth, @PRYear, @BPTL... — không tự đặt tên khác.
  4. Function đọc kỳ trước (retro / kỳ trước liền kề) → param dạng @Prev_<Mã> (template chuẩn). Engine set2 auto-parse để build dependency.
  5. Trạng thái mặc định = "Đề xuất" khi BA tạo mới. Dev chuyển sang "Đang phát triển" khi bắt tay code, "Đã có" khi review + deploy. "Chưa cần" giữ lại cho tương lai.
  6. SQL Server < 2019: scalar UDF không inline → performance kém khi gọi trong SELECT hàng loạt. Cân nhắc iTVF trả 1 cột + CROSS APPLY để tăng tốc, hoặc bật INLINE = ON nếu SQL 2019+.
